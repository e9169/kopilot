# Maintainers

This document lists project maintainers and the expectations for stewardship.

## Contributors

- GitHub Copilot (AI assistant)

## Responsibilities

- Review and merge pull requests
- Maintain release cadence and changelog quality
- Enforce security and code-quality standards
- Respond to security reports per SECURITY.md
- Keep CI and dependency management healthy

## Adding/Removing Maintainers

1. Open a pull request updating this file.
2. Obtain approval from an existing maintainer.
3. Announce the change in the release notes or repository discussions.

## Decision Process

- Day-to-day decisions are made by maintainers through PR review.
- Significant changes (architecture, releases) require at least two maintainer approvals.
